# FlowPrompt Studio v1.0.0

## 🎉 初始公开发布

我们很高兴地宣布 **FlowPrompt Studio 1.0.0** 正式发布！这是首个公开版本，包含完整的工程化提问工作流、Trae CN 深度集成、代码审计引擎和开发者能力评估系统。

**核心理念**：让提问更专业，让 AI 更懂你

---

## ✨ 核心功能

### 🎯 工程化提问工作台

通过**5 步结构化表单**引导开发者完成工程化提问：

1. **问题上下文** - 项目类型、技术栈、问题描述
2. **试错记录** - 记录已尝试方案及其结果
3. **核心问题定义** - 精准定义核心问题
4. **验收标准** - 可量化的验收指标
5. **功能/技术边界** - 明确功能范围和技术限制

**特色**：
- 📊 实时质量评分（100 分制）
- 💾 自动保存与草稿箱（支持 AES-256 加密）
- ✨ 一键生成标准化 Prompt
- 🔄 实时校验输入完整性

---

### 🤖 Trae CN 深度集成

无缝对接 Trae CN AI 助手：

- 🔍 **自动检测** - 智能识别 Trae CN 安装状态和版本
- 🚀 **一键发送** - 点击按钮即可发送 Prompt 到 Trae CN
- 📝 **上下文同步** - 保持对话连贯性，避免重复描述
- 🎨 **Prompt 格式化** - 根据 Trae CN 特性自动优化格式
- 🌐 **多 AI 支持** - 兼容 Copilot、Cursor 等主流 AI 助手

---

### 🔍 代码审计引擎

基于 Tree-sitter 的深度代码分析：

- 🌳 **AST 语法树分析** - 精确识别代码结构和模式
- 🌍 **10 种语言支持** - JavaScript/TypeScript/Python/Java/Go/C#/C++/Rust 等
- ⚠️ **实时问题检测** - 代码规范、潜在 bug、性能问题
- 💡 **智能修复建议** - 提供可执行的修复方案
- 📊 **问题分类** - 按严重程度分级展示

---

### 📊 开发者能力中心

全面评估开发者工程化能力：

- 📈 **CI 指数评估** - 综合评估工程化能力水平
- 🎯 **能力雷达图** - 多维度可视化展示能力分布
- 📊 **成长轨迹追踪** - 记录能力提升曲线

## 📖 快速开始

### 第一次使用

**1. 打开工作台**
- 点击左侧活动栏的 FlowPrompt Studio 图标
- 或按 `Ctrl+Shift+P` → "FlowPrompt: 打开工程化提问工作台"

**2. 填写表单**（5 步）

```
步骤 1/5: 问题上下文
- 选择项目类型（Web 前端/后端/移动应用等）
- 输入技术栈（React, Vue, Spring Boot 等）
- 详细描述问题背景

步骤 2/5: 试错记录
- 添加已尝试的方案
- 记录每个方案的结果（成功/失败）
- 粘贴错误日志和报错信息

步骤 3/5: 核心问题定义
- 用一句话定义核心问题
- 添加约束条件（性能、兼容性等）

步骤 4/5: 验收标准
- 添加可量化的验收指标
- 例如：响应时间 < 200ms, 支持 1000 并发

步骤 5/5: 功能/技术边界
- 明确功能范围（包含什么、不包含什么）
- 明确技术限制（浏览器版本、Node 版本等）
```

**3. 生成 Prompt**
- 点击"生成 Prompt"按钮
- 查看生成的标准化 Prompt
- 可手动编辑优化

**4. 发送至 AI**
- 点击"发送至 Trae CN"
- 自动打开 Trae CN 并发送 Prompt
- 开始 AI 对话

## 🐛 已知问题

### 轻微问题

1. **C#/C++/Rust 支持需手动安装**
   - **原因**: 这些语言使用频率较低，为减小包体积设为可选
   - **解决方案**: 按需安装对应依赖
   ```bash
   npm install tree-sitter-c-sharp  # C# 支持
   npm install tree-sitter-cpp      # C++ 支持
   npm install tree-sitter-rust     # Rust 支持
   ```

2. **ONNX AI 校准功能未默认启用**
   - **原因**: ONNX 运行时体积过大（210 MB）
   - **影响**: AI 校准功能使用纯 JS fallback 方案
   - **解决方案**: 按需安装
   ```bash
   npm install onnxruntime-node
   ```
